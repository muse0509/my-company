## Roadmap Creation Process

### Step 1: Vision Definition

**Question:**
- Where do we want to be in 12 months?

**Axis Pizza Vision:**
- 10K users
- Top 3 DeFi UX on Solana
- Multi-protocol aggregator
- Mobile-first experience

### Step 2: User Research

**Methods:**
1. User interviews（月2回）
2. Feedback analysis（Discord, X, App）
3. Usage data（127 users行動分析）
4. Competitor analysis

**Current Insights:**
- #1 Request: Mobile app
- #2 Request: Jupiter integration
- #3 Request: Portfolio tracking
- Pain Point: Liquidation risk（初心者）

### Step 3: Feature Ideation

**Brainstorm:**
- Product Lead主導
- CTO技術実現性確認
- Marketing Lead市場ニーズ確認
- Research Leadトレンド確認

**Output:**
- 20-30 feature ideas
- Rough effort estimates
- Initial priority感覚

### Step 4: RICE Prioritization

**For each feature:**
1. Reach（何人に影響？）
2. Impact（どれくらい改善？）
3. Confidence（確実性は？）
4. Effort（工数は？）

**Scoring:**
- RICE Score = (R × I × C) / E
- Top 10をロードマップに

### Step 5: Timeline Allocation

**Now（0-1 Month）:**
- RICE Top 3
- Quick wins（低Effort, 高Impact）

**Next（1-3 Months）:**
- RICE 4-7
- Medium complexity

**Later（3-12 Months）:**
- RICE 8-10
- Long-term bets

### Step 6: Review & Update

**Frequency:**
- Weekly: Now進捗確認
- Monthly: Next調整、Later追加
- Quarterly: Vision再評価

## Roadmap Adjustments

### When to Pivot

**Triggers:**
1. **User Feedback:**
   - Strong demand for new feature
   - Low adoption of planned feature

2. **Market Changes:**
   - Competitor launches similar feature
   - New protocol/tech emerges

3. **Technical Constraints:**
   - Implementation harder than expected
   - Dependency blockers

4. **Resource Changes:**
   - Team size変化
   - Funding状況

### How to Pivot

**Process:**
1. Identify trigger
2. Re-run RICE evaluation
3. Consult team（CTO, Marketing, Research）
4. Update roadmap
5. Communicate changes（internal + external）

**Example:**
"Pivoting from Dark Mode to Mobile App based on user feedback (50 requests vs 5). Mobile = 3x RICE score."
