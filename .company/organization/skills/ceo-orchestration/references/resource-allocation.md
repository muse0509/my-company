## Allocation Framework

### Step 1: Prioritize

**Use RICE scoring:**
```
Priority = (Reach × Impact × Confidence) / Effort

Example:
Project A: (1000 × 3 × 0.8) / 4 = 600
Project B: (100 × 5 × 1.0) / 1 = 500

Allocate to Project A first
```

**Factors:**
- Reach: How many affected?
- Impact: How significant? (1-5 scale)
- Confidence: How certain? (0-1)
- Effort: Person-weeks or $ cost

### Step 2: Allocate

**Budget allocation:**
- Check department budget
- Check monthly remaining
- Allocate to highest RICE score
- Reserve 20% for contingency

**Time allocation:**
- Estimate effort
- Check team capacity
- Assign to least loaded person
- Add buffer (×1.5 estimate)

**Token allocation:**
- Estimate token cost
- Check department token budget
- Optimize with COO if high
- Monitor daily burn

### Step 3: Monitor

**Track weekly:**
- Budget: Actual vs. allocated
- Time: Hours spent vs. estimated
- Tokens: Consumed vs. budgeted
- Output: Quality & completion %

**Adjust if:**
- Budget overrun >20%
- Time overrun >50%
- Token overrun >30%
- Quality below threshold

### Step 4: Optimize

**Monthly review:**
- Which projects over/under budget?
- Where to reallocate?
- What to cut?
- What to scale up?

**Continuous improvement:**
- Document lessons
- Update estimates
- Improve prioritization
- Reduce waste

## Reallocation Rules

### When to Reallocate

**Triggers:**
- Department consistently under budget (reallocate elsewhere)
- Department over budget (investigate & possibly increase)
- Strategic shift (move resources to priority)
- Emergency (tap contingency)

**Process:**
1. Identify need
2. Check contingency available
3. Identify low-priority spend
4. Reallocate
5. Document & monitor

### Emergency Allocation

**Contingency fund: 20% of monthly budget**

**Use for:**
- Unexpected opportunities
- Crisis management
- Critical bugs/issues
- Urgent priorities

**Approval:**
- <$500: CEO approves
- $500-$2K: CTO or COO review + CEO approves
- >$2K: Escalate to Muse

## Cost Optimization

### Regular Reviews

**Weekly:**
- Check burn rate vs. budget
- Identify overspend
- Quick fixes (pause non-essential)

**Monthly:**
- Review all subscriptions (cancel unused)
- Analyze department efficiency (cost per output)
- Optimize with COO (token costs)

**Quarterly:**
- Major budget review
- Reallocate based on performance
- Renegotiate contracts
- Strategic reallocation

### Optimization Patterns

**Pattern 1: Subscription Audit**
```
Monthly: Review all subscriptions
- Still needed? (Usage data)
- Cheaper alternative?
- Batch with other tools?
- Negotiate discount?

Savings potential: 20-30%
```

**Pattern 2: Freelancer vs. FTE**
```
Decision:
- <20h/week → Freelancer
- 20-40h/week → Consider FTE
- Specialized task → Freelancer
- Core competency → FTE

Flexibility: Freelancer > FTE
Cost long-term: FTE < Freelancer
```

**Pattern 3: Build vs. Buy**
```
Build when:
- Core differentiator
- Available time
- Long-term need
- Custom requirements

Buy when:
- Commodity function
- Time-critical
- Short-term need
- Standard requirements

Rule: Buy unless strategic.
```

### ROI Calculation

**Standard ROI:**
```
ROI = (Gain - Cost) / Cost × 100%

Example:
Marketing campaign:
- Cost: $1K
- Gain: 500 followers → 5 meetings → 1 investor → $50K
- ROI = (50000 - 1000) / 1000 = 4900%

Worth it!
```

**Calculate before spending >$500:**
- Expected gain?
- Probability of success?
- Expected ROI?
- Payback period?

**Approve if:**
- ROI >200% (spend $1, get $3 back)
- Payback <6 months
- Strategic value beyond $ ROI

## Approval Process

### Tier 1: Instant (<$100)

**Process:**
1. CEO decides
2. Execute
3. Log expense

**Example:** Subscribe to $49/month tool

### Tier 2: Quick (<$1K)

**Process:**
1. CEO decides
2. Brief rationale documented
3. Execute
4. Log expense
5. Weekly review

**Example:** Hire freelancer for $500 design work

### Tier 3: Review (<$10K)

**Process:**
1. Document need + options
2. Calculate ROI
3. Get CTO/COO input (if relevant)
4. CEO approves
5. Execute
6. Monitor closely

**Example:** $5K marketing campaign

### Tier 4: Strategic (>$10K)

**Process:**
1. Detailed proposal
2. ROI analysis
3. C-Suite review
4. CEO recommendation
5. **Escalate to Muse**
6. Muse approves
7. Execute
8. Report back

**Example:** Hire full-time engineer ($120K/year)
