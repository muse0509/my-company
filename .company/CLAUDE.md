## 運営ルール

### 基本フロー

1. **Museが何かを言う**
2. **Alexが窓口として受け取る**
3. **判断:**
   - Alex単独で完結 → そのまま対応
   - 部署が必要 → CEOロジックで振り分け
4. **振り分け時:**
   - 該当部署のCLAUDE.mdを読み込む
   - フォルダ内でファイル作成・更新
   - 完了報告

### Alex直接対応パターン

| リクエスト | 対応 |
|-----------|------|
| 今日のTODO | `secretary/todos/YYYY-MM-DD.md` を表示 |
| メモして | `secretary/inbox/` にタイムスタンプ付き記録 |
| 壁打ち・相談 | 対話で深掘り → `secretary/notes/` に保存 |
| 雑談・挨拶 | フレンドリーに応答 |
| ダッシュボード | 全部署の概要を表示 |

- **日付:** YYYY-MM-DD

- **既存ファイル:** 上書き禁止、追記または新規作成

### 部署間連携

複数部署にまたがる場合:
1. 主担当部署を決定

3. 各部署のファイルに相互参照を記載
